# 🚀 Hexo 常用命令手册

这是 Hexo 静态博客搭建与日常维护的常用命令汇总。

## 1. 核心流程（最常用）
这是你每次写完文章后，发布到 GitHub 所需的“三部曲”：

| 命令            | 简写     | 作用                                           |
| :-------------- | :------- | :--------------------------------------------- |
| `hexo clean`    | 无       | **清理缓存**：删除旧的静态文件 (public 文件夹) |
| `hexo generate` | `hexo g` | **生成静态文件**：将 Markdown 转换为 HTML      |
| `hexo deploy`   | `hexo d` | **部署**：将生成的静态文件推送到 GitHub        |

> **组合技（推荐）：**
> `hexo clean && hexo g && hexo d`  # 一次性完成清理、生成和上传

---

## 2. 文章管理
用于创建新文章或页面。

| 命令                               | 作用                                         |
| :--------------------------------- | :------------------------------------------- |
| `hexo new "标题"`                  | **新建文章**：在 `source/_posts` 下创建文章  |
| `hexo new page "标题"`             | **新建页面**：创建一个独立页面（如“关于我”） |
| `hexo publish [layout] <filename>` | 将草稿发布为正式文章                         |

---

## 3. 本地预览
在正式发布前，先在本地查看效果。

| 命令                  | 简写     | 作用                                                 |
| :-------------------- | :------- | :--------------------------------------------------- |
| `hexo server`         | `hexo s` | **启动本地服务器**：默认地址 `http://localhost:4000` |
| `hexo server -p 5000` | 无       | 更改端口号（如果 4000 被占用）                       |

---

## 4. 初始化与安装
通常只在更换电脑或首次搭建时使用。

| 命令                                   | 作用                                  |
| :------------------------------------- | :------------------------------------ |
| `npm install -g hexo-cli`              | 全局安装 Hexo 客户端                  |
| `hexo init [folder]`                   | 初始化博客文件夹                      |
| `npm install`                          | 在博客目录下安装所有必要的依赖包      |
| `npm install hexo-deployer-git --save` | 安装 Git 部署插件（发布 GitHub 必装） |

---

## 5. 进阶参数（Debug）
当遇到报错（如你之前的 YAML 报错）时，可以用来排查。

* `hexo g --watch`：监视文件变动，保存即自动重新生成。
* `hexo --debug`：开启调试模式，终端会输出详细的运行日志。

---

## 💡 小贴士：
- **YAML 格式**：文章开头的 `title:`, `tags:`, `date:` 后面**必须有一个空格**，否则会报错。
- **配置生效**：修改了 `_config.yml`（站点配置）后，建议先 `hexo clean` 再重新生成。
- **图片存放**：建议安装 `hexo-asset-image` 插件，或者将图片上传到图床（如 GitHub 仓库或阿里云 OSS）。
